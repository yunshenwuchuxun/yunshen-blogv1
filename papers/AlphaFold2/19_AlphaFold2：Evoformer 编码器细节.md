# AlphaFold2：Evoformer 编码器细节

> Evoformer 是 AlphaFold2 的核心编码器，它同时处理 MSA 表示和配对表示两个二维输入，通过 48 个 Transformer Block 提取氨基酸之间的深层关系。

## Evoformer 概述

![Evoformer 接收 MSA 表示和配对表示作为输入，经过 48 个 Block 精炼后输出](./解释_assets/media/image165.png)

**Evoformer** 是 Transformer 的一个变种，与标准 Transformer 的关键区别在于：

1. 输入不是单个序列（或图片块序列），而是**两个表示**同时作为输入
2. 输入的序列维度是**二维矩阵**，而非直接的一维序列

Evoformer 使用 48 个 Transformer Block，逐步提取和精炼不同氨基酸之间的关系。

## 行注意力：Row-wise Gated Self-Attention with Pair Bias

这是 Evoformer 中处理 MSA 表示的核心模块。

![Algorithm 7：MSA 行注意力伪代码，展示了配对偏置和门控机制](./解释_assets/media/image167.png)

具体步骤：

1. **层归一化**：对输入的 MSA 特征 $\mathbf{m}_{si}$ 进行 LayerNorm，稳定后续计算
2. **生成 Q、K、V**：将归一化后的 MSA 特征线性投影，生成查询 $\mathbf{q}$、键 $\mathbf{k}$ 和值 $\mathbf{v}$
3. **提取配对偏置**：从配对表示 $\mathbf{z}_{ij}$ 中提取并线性投影，生成偏置 $\mathbf{b}$（几何提示）
4. **计算注意力分数**：

$$a_{sij}^h = \text{softmax}_j \left( \frac{1}{\sqrt{c}} \mathbf{q}_{si}^{h\top} \mathbf{k}_{sj}^h + b_{ij}^h \right)$$

在 Q 和 K 的相似度基础上，**直接加上配对表示的偏置** $b$ 作为引导，然后归一化

5. **门控信号**：从 MSA 特征中计算 0 到 1 之间的门控信号 $\mathbf{g}$，控制信息流通量

$$\mathbf{g}_{si}^h = \sigma(\text{Linear}(\mathbf{m}_{si}))$$

6. **加权聚合**：用注意力分数对 $\mathbf{v}$ 加权求和，并用门控信号逐元素调节输出强度

$$\mathbf{o}_{si}^h = \mathbf{g}_{si}^h \odot \sum_j a_{sij}^h \mathbf{v}_{sj}^h$$

7. **输出投影**：将所有注意力头的输出拼接，通过线性层整合
8. 返回精炼后的 MSA 特征

## 列注意力：Column-wise Gated Self-Attention

与行注意力过程类似，但**没有加入配对偏置**。列注意力沿序列维度（不同同源序列之间）进行信息混合，捕捉不同物种之间在同一位置上的变异模式。

自注意力主要负责混合不同元素之间的信息，而真正做信息提取和深度加工的是后续的 MLP。

## Transition（前馈网络）

Transition 模块是一个标准的前馈网络，负责对信息进行深度加工和提炼：

1. **第一个线性层（升维）**：将 128 维的输入向量 $\mathbf{x}$ 通过权重矩阵 $W_1$（维度 $128 \times 512$）扩展到 512 维
   - 计算：$\mathbf{h} = \mathbf{x} \cdot W_1 + b_1$
2. **非线性激活（ReLU）**：对 512 维的向量进行非线性处理
3. **第二个线性层（降维）**：将 512 维的向量通过权重矩阵 $W_2$（维度 $512 \times 128$）压缩回 128 维

整体流程：输入 -> 升维 -> 非线性计算 -> 降维 -> 输出。

## Outer Product Mean（外积均值）

Outer Product Mean 是一个专门的"共进化信号转换器"，将一维的序列特征"升维"为二维的成对关系特征。

### Mean（均值）：提炼进化共识

- **输入**：经过行、列注意力精炼后的 MSA 表示，形状为 $(S, L, C)$
  - $S$ = 序列数量（如数千条同源序列）
  - $L$ = 蛋白质长度（氨基酸数量）
  - $C$ = 特征维度
- **操作**：沿 $S$ 维度进行**平均化**，将成百上千条同源序列的信息"压缩"为一条"共识序列"
- **目的**：去除单条序列的噪声，提炼进化过程中最稳定、最普遍的特征模式

### Outer Product（外积）：创建关系矩阵

- **输入**：经过 mean 操作后的 $(L, C)$ 特征矩阵，第 $i$ 个位置的特征向量记为 $\mathbf{m}_i$
- **操作**：对任意位置对 $(i, j)$，计算 $\mathbf{m}_i$ 和 $\mathbf{m}_j$ 的外积
  - 若 $\mathbf{m}_i$ 是 $(C, 1)$ 的列向量，$\mathbf{m}_j$ 是 $(1, C)$ 的行向量，外积生成 $(C, C)$ 的矩阵
  - 这个矩阵编码了位置 $i$ 和 $j$ 之间所有特征的完整配对互动信息
- **意义**：如果 $\mathbf{m}_i$ 和 $\mathbf{m}_j$ 在某些特征维度上都表现出强烈信号（例如都倾向于带正电荷的氨基酸），外积矩阵中对应位置的数值就会很大，产生强烈的"关联"信号

Outer Product Mean 模块将进化数据库中"哪些氨基酸倾向于协同变化"的信息，转化为"哪些氨基酸在空间上可能相互接触"的预测，并更新到配对表示中。

## Triangle Self-Attention（三角自注意力）

### Around Starting Node

三角自注意力是用于执行几何推理的模块。它将**边**（而非节点）作为处理对象，从一个固定"起始节点"的视角出发，利用自注意力机制聚合所有相关的三角信息。

具体过程：

1. **收集"边"信息**：模型选择一个起始节点 $i$，从配对表示中取出两条关键的边信息——边 $ij$（$i$ 和 $j$ 之间的空间关系）与边 $ik$（$i$ 和 $k$ 之间的空间关系）
2. **"边"作为注意力输入**：在标准自注意力中输入是节点（如单词），而在三角自注意力中输入变成了一系列**边**。对于起始节点 $i$，模型会把它和所有其他节点 $k$（$k = 1, 2, \ldots, L$）连接成的一整行"边" $(i,1), (i,2), \ldots, (i,L)$ 作为输入序列
3. **计算注意力**：自注意力在这条由"边"组成的序列上进行计算
   - Query 来自于边 $ij$
   - Key 和 Value 来自于边 $ik$
   - 模型计算 Query($ij$) 和 Key($ik$) 的相似度，得到注意力分数，表示"从 $i$ 的视角看，边 $ij$ 和边 $ik$ 有多相关"
   - 然后用这个分数去加权求和所有的 Value($ik$)
4. **更新目标**：加权求和的结果用来更新边 $jk$ 的信息

AlphaFold2 中还有方向相反的模块 **Triangle Self-Attention around Ending Node**，原理完全相同，只是观察视角从 $(i, j), (i, k)$ 变成 $(j, i), (k, i)$，确保信息能够双向流动。

## 编码器的输出

经过 48 层 Evoformer 精炼后，编码器输出两个关键表示：

### 精炼后的 MSA 表示

- 形状与输入相同的三维张量 $(S, L, C)$，但内部特征已被彻底重写和优化
- 从中提取目标序列对应的那一行，作为**单序列表示（Single Representation）**，形状为 $(L, C)$，描述每个氨基酸自身的最终状态和所处环境

### 精炼后的配对表示

这是编码器**最重要的输出**。

- 形状为 $(L, L, C')$ 的三维张量，可以想象成一张 $L \times L$ 的二维"地图"
- 地图上每个点 $(i, j)$ 存储一个高维向量，包含：
  - 氨基酸 $i$ 和 $j$ 在三维空间中的**距离**（如 8.5 埃）
  - $i$ 和 $j$ 的**相对朝向**（平行排列还是垂直排列）
- 这是模型对蛋白质三维结构高度自信的**几何蓝图**

| | 精炼后的配对表示 | 精炼后的单序列表示 |
|---|---|---|
| 回答的问题 | 氨基酸 $i$ 和 $j$ 的关系是怎样的？ | 氨基酸 $i$ 自身的最终状态和所处环境是怎样的？ |
| 数据结构 | $L \times L$ 的关系地图 | 长度为 $L$ 的状态档案列表 |
| 核心信息 | 距离、方向等几何关系 | 局部结构环境（螺旋/折叠）等节点属性 |
| 比喻 | 案件的"人物关系图" | 每个核心人物的"详细个人档案" |
