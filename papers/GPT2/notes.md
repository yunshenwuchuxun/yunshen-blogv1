# GPT-2：语言模型即无监督多任务学习器

> GPT-2 的核心贡献在于证明：当语言模型足够大、训练数据足够丰富时，无需任何微调，仅凭 **Prompt**（提示）就能零样本完成多种任务。这一发现开创了从"改变模型"到"改变输入"的范式转变。

## 从微调范式到 Prompt 范式

在 GPT-2 之前，主流做法是"**预训练 + 微调**"（Pre-training + Fine-tuning），以 BERT 和 GPT-1 为代表：

1. **预训练**（Pre-training）：用海量无标签文本（如维基百科）训练一个通用基础模型，让它学会语言的语法、事实知识和常识推理。
2. **微调**（Fine-tuning）：当需要解决特定任务（如情感分析）时，研究者需要准备该任务的**有标签数据集**，在预训练模型之上添加一个任务专属的分类"头"，再用标注数据训练模型权重以适配特定任务。

这个过程费时费力，且需要大量标注数据。

**GPT-2 的创新**则完全不同：当模型在足够多样、高质量的数据（WebText）上训练后，它内部已经隐式学会了各种任务的模式。我们不需要修改参数，只需通过 Prompt 来"激活"或"引导"它执行特定任务。这种能力被称为**零样本学习**（Zero-Shot Learning）。

## Prompt 的具体实现与示例

GPT-2 团队发现，WebText 训练数据中天然存在各种任务的"模式"：

- **翻译**：数据中包含类似 `"The following is an English sentence 'sea otter' translated to French is 'loutre de mer'"` 的文本。因此，只要给模型输入 `translate to french: sea otter =>`，模型为了让文本"自然"，就会续写出 `loutre de mer`。
- **文章摘要**：数据中的新闻文章或论坛帖子常在结尾带有 `TL;DR:`（Too Long; Didn't Read）总结。因此，给模型输入一篇长文章，然后在末尾加上 `TL;DR:`，模型就会"认为"接下来应生成一段摘要。
- **阅读理解**：将上下文文章和问题组合成 `{文章内容} Q: {问题} A:` 的格式，模型就会在 `A:` 后面续写出答案。

## 为什么 Prompt 是重大创新

**从"改变模型"到"改变输入"**：实现了根本性转变——从为每个任务修改和重新训练模型（改变参数），转变为使用同一个固定模型，只通过改变输入文本（设计 Prompt）来解决问题。这极大地提升了模型的通用性和易用性。

**零样本学习能力的验证**：GPT-2 能完成这些任务而无需任何标注样本进行训练，仅依靠预训练数据中学到的模式就能"举一反三"。这是零样本概念在大型语言模型上的首次大规模成功验证。

**统一所有 NLP 任务**：Prompting 范式将分类、生成、翻译、问答等看似不同的 NLP 任务统一到了一个极简框架：**文本输入 -> 文本输出**（Text-to-Text）。

## 微调范式 vs. Prompt 范式对比

| 对比维度 | 预训练-微调范式 | Prompt 范式 |
|:---:|:---|:---|
| 核心思想 | **模型为本**：改变模型去适配特定任务 | **任务为本**：改变输入（提示）来引导通用模型 |
| 任务适应方式 | 在预训练模型上用特定任务的标注数据进行二次训练（微调），更新模型权重 | 保持预训练模型参数不变，通过设计巧妙的文本提示（Prompt）来引导模型执行任务 |
| 数据需求 | 需要为每个任务准备中到大规模的标注数据集 | 无需或只需极少量样本（Zero-Shot 或 Few-Shot） |
| 模型使用方式 | 模型参数在微调过程中被修改，每个任务最终对应一个独立的模型副本 | 模型参数保持不变（冻结），所有任务共享同一个通用大模型 |
| 参数效率 | 低——每个新任务都需要存储一套独立的模型参数，成本高 | 高——只需一个模型即可应对所有任务，无需额外存储 |
| 灵活性/通用性 | 差——切换任务等于切换模型，难以快速适应没有标注数据的新任务 | 强——可以快速、动态地解决新任务，只需设计新的 Prompt |
| 性能表现 | 在有充足标注数据的情况下，通常能在特定任务上达到非常高的性能 | 性能高度依赖模型规模和 Prompt 质量；对于超大模型，在很多任务上能达到甚至超过微调模型的性能 |
| 代表模型 | BERT、GPT-1、RoBERTa 等 | GPT-2（开创者）、GPT-3、GPT-4、ChatGPT 等 |
| 通俗比喻 | **专科培养**：为了让学生擅长数学就送去数学系专门训练，每个学科都需要专门培养 | **通才教育**：培养一个知识渊博的通才（大模型），只需用清晰的语言给他下达指令（Prompt），他就能解决各种问题 |

## 小结

GPT-2 中 Prompt 的提出，并非发明了一个词，而是开创并证实了一种全新的、与大模型交互和利用其能力的方式。它将模型的应用从繁琐的、任务特定的微调中解放出来，为后来 GPT-3 的**情境学习**（In-context Learning）、InstructGPT 的**指令遵循**（Instruction Following）以及 ChatGPT 等产品奠定了至关重要的基础。
