# MoCo：Momentum Contrast for Unsupervised Visual Representation Learning

> MoCo 是自监督视觉表示学习的里程碑工作。它将对比学习建模为字典查询问题，通过动态队列和动量编码器两大核心机制，构建了大型且一致的负样本字典，显著缩小了自监督学习与监督学习之间的差距。

## 对比学习基本概念

**对比学习**是一种自监督学习方法，核心思想是学习区分相似和不相似样本的表示。它通过构建正样本对和负样本对，训练模型在特征空间中将正样本拉近、负样本推远，无需人工标注标签。

- **正样本拉近**：对同一张猫的照片做两种数据增强（裁剪、旋转等），得到两个视图。对比学习的目标是让这两个视图的特征表示非常接近，因为它们本质上是同一只猫。
- **负样本推远**：当模型同时看到猫和狗的照片时，应产生两个在特征空间中相距较远的向量，表明它们是不同的概念。

## 研究背景与动机

自监督视觉表示学习旨在无需人工标注的情况下学习有效的视觉特征。现有方法主要分为生成式和判别式两类，其中对比学习作为**判别式方法**展现出巨大潜力。

对比学习可以从字典查询的视角理解：由 $x_1$ 生成的增强视图 $x_1^1$ 作为 anchor，$x_1^2$ 是 positive，其他样本 $x_2 \dots x_N$ 是 negative。在特征空间里，目标是让 $x_1^1$ 和 $x_1^2$ 尽可能相近，$x_1^1$ 与 $x_2 \dots x_N$ 尽可能远离。

换一个视角，可以将 $f_1^1$ 看作字典的 **query**（查询），$f_1^2, f_2 \dots f_N$ 看作 **key**，让 query 尽可能匹配正样本的特征。在 MoCo 论文中，记号对应关系为：$x_1^1 \to x^q$，$x_1^2 \to x^k$，$f_1^1 \to q$，$f_1^2 \to k_0$，$f_2 \to k_1 \dots$。其中 $q$ 使用一个编码器，$k_0 \dots k_N$ 使用另一个编码器。

MoCo 旨在构建一个大型且动态更新的字典，同时保持表示的一致性，从而提升自监督视觉表示学习的效果。

## 核心设计

### 动态队列机制

对比学习的效果很大程度上取决于**负样本的数量和质量**。大型字典可以提供更多负样本，但如何保持这些样本表示的一致性是关键挑战。

MoCo 设计了一个动态队列（queue）结构作为字典：新的 mini-batch 编码后的键入队，最早入队的键出队，保持字典大小固定。这种设计将字典大小与 mini-batch 大小解耦，允许使用大量负样本而不受 GPU 内存限制。

队列中被移除的键会被完全丢弃，不会被保存或重用，不再参与损失计算，其内存空间被新入队的键覆盖。移除旧键的原因有两个：

1. **保持队列大小固定**：队列有固定的大小限制（如 65536），每当新键入队时必须有旧键出队
2. **保持表示的"新鲜度"**：随着模型训练，特征表示会逐渐变化，太旧的键可能与当前模型的表示标准不一致，移除最旧的键有助于保持队列中表示的相对一致性

### 一致性问题

在对比学习中，**表示一致性**问题指的是：不同时间点编码的特征向量是否使用相似的编码标准。

在 MoCo 中，队列存储的是不同时间点编码的键。如果编码器参数变化太快，早期入队的键和最近入队的键可能使用了非常不同的编码标准。这种不一致会导致对比学习效果变差，因为负样本之间的差异可能来自编码标准的变化，而非真正的语义差异。

### 动量编码器

为解决一致性问题，MoCo 引入了**动量编码器**（momentum encoder）。查询编码器参数 $\theta_q$ 通过反向传播正常更新，而键编码器参数 $\theta_k$ 通过动量更新：

$$\theta_k = m \cdot \theta_k + (1 - m) \cdot \theta_q$$

其中 $m$ 是动量系数（通常设为 0.999）。更新流程如下：

1. **查询编码器更新**：处理当前批次的查询样本，通过反向传播和梯度下降正常更新 $\theta_q$，这是标准的深度学习参数更新过程
2. **键编码器更新**：不通过反向传播更新，而是使用上面的动量更新公式，本质上是查询编码器参数的"滞后版本"

由于 $m$ 接近 1，键编码器参数缓慢演变，从而提供更一致的表示。

## 代理任务

**代理任务**（Pretext Task）是自监督学习中为学习有用的特征表示而设计的辅助任务。这些任务不需要人工标注标签，而是利用数据本身的结构创建"伪标签"。

### 核心思想

- **自动生成监督信号**：从数据本身自动创建标签，无需人工标注
- **学习通用表示**：通过解决代理任务，模型学到对下游任务有用的特征
- **迁移学习**：学到的表示可以迁移到真正关心的下游任务（如分类、检测）

### 代理任务与下游任务的关系

采用"预训练-微调"范式：先在代理任务上预训练模型，然后在下游任务上微调，或直接提取特征用于下游任务。代理任务本身的性能不是最终目标，关键是在下游任务上的表现。

### 经典案例

| 方法 | 任务类型 | 伪标签生成 | 学习目标 |
|------|---------|----------|---------|
| MoCo | 实例判别（对比学习） | 同一图像的不同增强视图为正样本，不同图像为负样本 | 使同一图像的不同视图在特征空间接近，不同图像远离 |
| BERT | 掩码语言建模（MLM） | 随机掩码 15% 的词，让模型预测原词 | 理解词的上下文关系和句子间关系 |
| MAE | 掩码图像建模 | 随机掩码 75% 的图像 patches，让模型重建原始像素 | 理解图像的语义和结构信息 |

### 自监督学习的两大支柱

- **代理任务**：提供"学什么"的框架，创造学习的情境，设计合适的代理任务可引导模型学习有意义的特征表示
- **目标函数**（Objective Functions）：定义"如何学"的标准，为模型学习提供可优化的数值目标，通过梯度下降等方法指导模型参数更新

## MoCo 整体架构

MoCo 包含三个核心组件：

- **查询编码器**：处理当前 mini-batch 的查询样本，参数通过反向传播更新
- **键编码器**：处理正负样本，参数通过动量更新
- **队列**：维护最近编码的键，作为负样本库

两个编码器架构相同（如 ResNet-50），但参数更新方式不同。

## 损失函数

### 为什么不直接用 Softmax + Cross Entropy

在对比学习中，每一张图片都被当作独立的类别。以 ImageNet 为例，有 100 万张图片，如果对 100 万个类做 softmax 归一化，计算复杂度极高。

### InfoNCE Loss

**InfoNCE Loss**（对比学习损失）的一般形式为：

$$\mathcal{L}_{\text{InfoNCE}} = -\log \frac{\exp(\text{sim}(q, k^+))}{\exp(\text{sim}(q, k^+)) + \sum_{i=1}^{K} \exp(\text{sim}(q, k_i^-))}$$

MoCo 中使用点积作为相似度度量：

$$\mathcal{L}_{\text{InfoNCE}} = -\log \frac{\exp(q \cdot k^+ / \tau)}{\exp(q \cdot k^+ / \tau) + \sum_{i=1}^{K} \exp(q \cdot k_i^- / \tau)}$$

### 温度参数

**温度参数** $\tau$ 控制损失函数的形状：

- $\tau$ 较大时，损失函数较平滑，表征学习更稳健，但可能拉不开样本差距
- $\tau$ 较小时，softmax 更"尖锐"，对最高相似度的项更加敏感

## 三种对比学习架构对比

### 端到端方法

在端到端对比学习（如 SimCLR）中，mini-batch 大小本身就是负样本集合的来源，等效于字典大小。字典一致性较高，但受硬件限制字典不能做很大。

### Memory Bank 方法

关注字典的"大"，牺牲部分一致性。字典大小远大于 mini-batch，使用独立的负样本存储库。但由于 key 的表征在不同时间点更新，一致性较差。

### MoCo 方法

MoCo 统一了端到端方法和 Memory Bank 方法的优点：通过动态队列实现大字典，通过动量编码器保持一致性。

## 训练伪代码与实现细节

> 以下内容深入解读 MoCo 的训练伪代码（Algorithm 1），涵盖数据增强、正负样本构建、编码过程、损失计算和动量更新的完整流程。

### 训练伪代码

伪代码中有两个关键设计值得注意：

- `labels = zeros(N)`：因为正样本始终放在 logits 的第 0 列，所以标签全为 0
- `CrossEntropyLoss(logits, labels)`：目标是让正样本得分最大，本质上是将对比任务转化为 softmax 多分类任务的实现手法
- `k = k.detach()`：生成一个新张量，它共享数据但不参与梯度计算，确保梯度不会回传到键编码器

### 数据增强与正负样本构建

**正样本对构建：** MoCo 采用**实例判别**任务，将同一图像的不同增强视图作为正样本对：

1. 对每张图像应用两次随机数据增强（随机裁剪、水平翻转、颜色抖动等）
2. 一个视图通过查询编码器，另一个通过键编码器
3. 两个视图互为正样本

**负样本选择：** 队列中除正样本外的所有样本都作为负样本。这些负样本来自之前的 mini-batch，由动量编码器编码，保持了表示的一致性。

### 编码过程

1. 图像经过标准的 ResNet 主干网络
2. 最后的全局平均池化层后接一个 MLP 投影头（将特征映射到低维空间）
3. 最终特征进行 L2 归一化，得到单位长度的表示向量

### 关键超参数

| 超参数 | 值 | 说明 |
|--------|------|------|
| 动量系数 $m$ | 0.999 | 控制键编码器参数的更新速度 |
| 温度参数 $\tau$ | 0.07 | 控制 softmax 分布的尖锐程度 |
| 队列大小 $K$ | 65536 | 负样本数量 |
| 特征维度 | 128 | 投影头输出维度 |

### 理论基础与联系

**与噪声对比估计的联系：** MoCo 的对比损失可视为**噪声对比估计**（NCE）的一种形式，目标是区分正样本分布和噪声分布。

**与其他对比学习方法的关系：** MoCo 统一了端到端方法和 Memory Bank 方法的优点，同时克服了它们的局限性：

- 相比端到端方法：MoCo 支持更大的字典
- 相比 Memory Bank 方法：MoCo 提供更一致的表示

MoCo 通过对比学习隐式地学习了数据分布的特征，使得相似实例在特征空间中靠近、不相似实例远离，与信息最大化理论相符。这种设计使 MoCo 在多种下游任务中展现出接近甚至超过监督预训练的性能，为自监督视觉表示学习开辟了新的方向。
