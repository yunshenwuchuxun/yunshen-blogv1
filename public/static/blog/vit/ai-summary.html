An Image is Worth 16x16 Words:
Transformers for Image Recognition at Scale

作者：Alexey Dosovitskiy, Lucas Beyer, Alexander Kolesnikov 等

核心摘要：
尽管 Transformer 已经成为自然语言处理（NLP）领域的事实标准，但在计算机视觉（CV）中，它的应用仍然受到限制（通常作为 CNN 的辅助组件）。本文证明了这种对 CNN 的依赖并非必要：一个纯粹的 Transformer 模型，直接应用于图像切块（Image Patches）序列，在图像分类任务上就能表现得极好。当在大规模数据集（如 JFT-300M）上进行预训练后，Vision Transformer (ViT) 以更少的计算资源达到了惊人的 SOTA 表现。

2. 研究动机与问题背景 (Motivation)

痛点与现状的局限性

  • CNN 主导统治： 在 CV 领域，从 AlexNet 到 ResNet，由于卷积操作天然自带的“归纳偏置”（Inductive Bias，如平移等变性和局部性），CNN 一直是绝对的主流。
  • Transformer 跨界遇阻： 随着 BERT/GPT 展现出超凡的扩展能力（Scalability），大量研究尝试将 Attention 机制引入 CV。但先前的尝试大多是：用 Attention 增强 CNN（如 Non-local block），或者用极其复杂的局部/轴向注意力替换卷积。这些方法虽然理论有效，但在现代硬件（TPU/GPU）上效率极低，难以像大语言模型那样大规模扩展。

本文试图解决的核心问题

我们能否以最少量的架构修改，将标准 NLP 领域的 Transformer 直接应用到图像上？
换句话说，如果打破 CNN 长期以来的垄断，剥离掉预设的“图像归纳偏置”，仅仅依赖大规模数据驱动，纯 Transformer 能否看懂图片？

研究的重要性 (Significance)

这是计算机视觉走向“大统一”模型架构的开山之作。它证明了：在大规模数据面前，模型容量和扩展性比人类手工设计的“局部性先验”更加重要。ViT 的出现彻底改变了 CV 领域的发展轨迹，催生了后续的 Swin Transformer、MAE 等一系列革命性工作。

3. 数学表示与建模 (Formulation & Modeling)

ViT 的设计哲学是：尽可能贴近原始的 NLP Transformer 架构（Vaswani et al., 2017），这样就可以直接开箱即用经过高度优化的 NLP 架构实现。

图像被切分为固定大小的 patches，线性投影后加上位置编码，送入标准 Transformer Encoder，最后通过一个专门的 [class] token 输出分类结果。

3.1 核心变量定义

符号 含义 维度说明
$x$原始输入图像$\mathbb{R}^{H \times W \times C}$
$x_p$展平后的图像切块序列 (Patches)$\mathbb{R}^{N \times (P^2 \cdot C)}$
$(P, P)$每个切块的分辨率通常为 16x16 或 32x32
$N$切块数量（序列长度）$N = HW / P^2$
$D$Transformer 的隐层维度贯穿所有层的常量（如 768）

3.2 数学形式化与前向传播全流程

Step 1：图像切块与线性投影 (Patch Embedding)

标准 Transformer 接收 1D 词向量序列，因此我们将 2D 图像重塑为展平的 2D 切块序列。接着通过一个可学习的线性投影矩阵 $E \in \mathbb{R}^{(P^2 \cdot C) \times D}$ 将其映射到 $D$ 维。

Step 2：拼接 Class Token 与添加位置编码 (Prepending & Positional Embedding)

借鉴 BERT，在序列开头加入一个可学习的特殊分类向量 $x_{class}$。为了保留空间信息，将标准的可学习 1D 位置编码 $E_{pos} \in \mathbb{R}^{(N+1) \times D}$ 加到切块特征上。初始状态 $z_0$ 表示为：

$$ z_0 = [x_{class}; x^1_p E; x^2_p E; \dots; x^N_p E] + E_{pos} $$

Step 3：Transformer Encoder 计算

Transformer 编码器包含 $L$ 层交替的多头自注意力块（MSA）和多层感知机块（MLP）。在每个块之前应用层归一化（LayerNorm, LN），并在之后使用残差连接。MLP 包含两层以及 GELU 激活函数：

$$ z'_l = \text{MSA}(\text{LN}(z_{l-1})) + z_{l-1}, \quad l = 1 \dots L $$ $$ z_l = \text{MLP}(\text{LN}(z'_l)) + z'_l, \quad l = 1 \dots L $$

Step 4：分类头输出 (Classification Head)

经过 $L$ 层处理后，提取序列第一个位置（即 `[class]` token 对应的位置）的特征 $z^0_L$，再通过一次 LayerNorm 得到图像的最终表征 $y$，送入分类头（全连接层）：

$$ y = \text{LN}(z^0_L) $$
核心逻辑伪代码 (PyTorch 风格) 
# Input x: [batch_size, channels, height, width]
patches = extract_patches(x, patch_size=16) # shape: [B, N, P*P*C]
patch_embeds = linear_projection(patches)   # shape: [B, N, D]

# Prepend class token
cls_tokens = repeat(cls_token_param, B)     # shape: [B, 1, D]
seq = concat([cls_tokens, patch_embeds], dim=1) # shape: [B, N+1, D]

# Add positional embeddings
seq = seq + pos_embedding_param             # shape: [B, N+1, D]

# Transformer Encoder
for i in range(L):
    seq_norm = LayerNorm(seq)
    seq = MSA(seq_norm) + seq
    seq_norm = LayerNorm(seq)
    seq = MLP(seq_norm) + seq

# Extract class token output
img_repr = LayerNorm(seq[:, 0])             # shape: [B, D]
logits = Linear(img_repr)                   # shape: [B, num_classes]

4. 实验设置与复现细节 (Experiments & Reproducibility)

4.2 数据集与预处理

  • 预训练数据集：ImageNet-1k (1.3M)、ImageNet-21k (14M)、JFT-300M (303M 谷歌内部非公开高分辨率数据集)。
  • 下游评估数据集：ImageNet、ImageNet-ReaL、CIFAR-10/100、Oxford-IIIT Pets、Oxford Flowers-102 以及 VTAB (19个任务基准)。
  • 分辨率调整技术：通常在更大分辨率下微调能带来性能提升。当输入分辨率变大但 patch size 保持不变时，序列长度 $N$ 增加。此时使用 2D 插值（根据原始图像中的坐标）来对预训练的位置编码 $E_{pos}$ 进行插值扩展。这也是 ViT 唯一手动注入图像 2D 拓扑先验的地方。
训练超参数细节 (基于 Appendix Table 3 & 4)
  • 预训练 (Pre-training)：
    • 优化器：Adam ($\beta_1=0.9, \beta_2=0.999$)。这是个反直觉的选择，因为 ResNet 传统使用 SGD。作者发现在他们的设定下 Adam 对 ResNet 也有提升。
    • Batch Size：4096。
    • Weight Decay：0.1 (很高，对迁移学习有益)。
    • Learning Rate：基础学习率约 $8 \times 10^{-4}$，采用 linear warmup (10k steps) + linear decay 策略。
    • Dropout：在直接从 ImageNet 头开始训练时设为 0.1，用大容量数据集时不使用 Dropout (0.0)。
  • 微调 (Fine-tuning)：
    • 优化器：SGD with momentum (0.9)。
    • Batch Size：512。
    • Learning Rate：网格搜索 $\{0.001, 0.003, 0.01, 0.03\}$，cosine 衰减策略。
    • 微调分辨率：通常高于预训练。比如 ViT-L/16 在 512 分辨率微调，ViT-H/14 在 518 分辨率微调。同时结合了 Polyak & Juditsky (EMA) 平均。

5. 实验结果与核心结论 (Results)

5.1 主流基准 SOTA 对比

将使用 JFT-300M 预训练的 ViT 与 SOTA 的卷积网络（如谷歌自家的 BiT-L 和基于 EfficientNet 架构的 Noisy Student）对比。测试指标为分类 Top-1 Accuracy (%)：

模型与预训练集 ImageNet ImageNet ReaL CIFAR-100 VTAB (19 tasks) TPUv3 Core-days
Ours-JFT (ViT-H/14) 88.55 $\pm$ 0.04 90.72 $\pm$ 0.05 94.55 $\pm$ 0.04 77.63 $\pm$ 0.23 2.5k
Ours-JFT (ViT-L/16) 87.76 $\pm$ 0.03 90.54 $\pm$ 0.03 93.90 $\pm$ 0.05 76.28 $\pm$ 0.46 0.68k
BiT-L (ResNet152x4) 87.54 $\pm$ 0.02 90.54 93.51 $\pm$ 0.08 76.29 $\pm$ 1.70 9.9k
Noisy Student (EfficientNet-L2) 88.4 / 88.5 90.55 - - 12.3k

核心发现： ViT-H/14 不仅在全面打败了极其强大的 CNN 基线，其预训练所需算力仅仅是 BiT-L 的 1/4，是 Noisy Student 的 1/5。这是非常惊人的效率革命。

5.2 预训练数据规模的影响 (Data Requirements)

在 ImageNet (1.3M) 上训练时，ResNet 碾压 ViT。但在 JFT (300M) 上，ViT 大比分超越 ResNet。

论文的一大金句是：“Large scale training trumps inductive bias” (大规模训练胜过归纳偏置)。
当在 ImageNet（中等规模）上训练时，由于 ViT 缺乏 CNN 那种自带的平移不变性和局部性，模型很容易过拟合，性能逊于 ResNet。然而，一旦数据量达到千万级（ImageNet-21k）乃至数亿级（JFT-300M），直接从海量数据中“学习”到的规律就能完美填补这些先验缺陷，并因为没有受限于人为设定的感受野而爆发出更强上限。

5.3 内部表征可视化：ViT 如何“看”图？

  • 位置编码提取了空间几何： 计算学到的位置编码的余弦相似度，发现模型自发地学到了 2D 的行、列网格结构（同一行/列的 patch 有更高的相似度）。这意味着强加 2D 位置编码是多余的。
  • 注意力距离 (Attention Distance)： 这是 ViT 里的“感受野”。观察发现，在底层网络中，某些注意力头只看很近的像素（类似小卷积核），而另一些头直接看全图；随着网络加深，注意力距离整体上升，高层几乎完全在做全局信息整合。

6. 审稿人视角的深度点评 (Critical Review)

🌟 创新与优点

  • 极简之美，大道至简： 直接把图像切成 16x16 的词汇送进最原始的 NLP Transformer，没有任何花哨的魔改架构（如局部注意力、轴向注意力等）。它在硬件加速器上执行效率极高，证明了只要数据管够，“通用架构”可以抹平模态差异。
  • 惊人的 Scalability (扩展性)： 论文不仅跑赢了 SOTA，更重要的是展示了一条清晰的“性能与算力成正比且尚未饱和”的 Scaling Law 曲线。

⚠️ 局限与不足

  • 对算力和数据的饥渴度极高： 普通实验室根本没有 JFT-300M 这种规模的数据。在标准的百万级 ImageNet 数据下，原版 ViT 是打不过 ResNet 的（这也是后来为何出现了 DeiT 通过蒸馏来抢救 Data Efficiency 的原因）。
  • 高分辨率下的二次方计算灾难： Attention 机制的复杂度与序列长度 $N$ 成二次方 $O(N^2)$。面对医疗影像或遥感这种超高分辨率图像，如果强行切成 16x16 的 patch，序列长度会爆炸导致 OOM（这催生了后来的 Swin Transformer 使用局部窗口）。
  • 密集预测任务的短板： 论文主要做的是分类任务，输出只是一个 `[class]` token。如果在当时做目标检测或像素级分割，单一尺度的特征图显得十分笨拙。

🚀 未来改进方向 (The Future)

历史已经证明，ViT 的缺点都被后人漂亮地解决了：

  1. 针对数据饥渴，引入自监督学习体系：如 MAE (Masked Autoencoders) 将 NLP 中的 BERT 掩码预测任务完美带到 CV 领域，成为 ViT 最佳拍档。
  2. 针对超大分辨率与密集任务，引入层级结构与局部注意力机制：如 Swin Transformer 重拾了 CNN 的多尺度特征设计。

One More Thing

一个极易被忽略的实验细节：CNN 的优化器选择玄学

在附录 D.1 中，作者提到了一个非常有趣的反直觉现象。传统观念里，无论在 PyTorch 官方教程还是何恺明的原论文中，训练 ResNet 的标配永远是 SGD with momentum。

但在这篇论文的 JFT 预训练阶段，作者给所有的基线 ResNet（即 BiT）使用的是 Adam 优化器。为什么？作者做了一个对照实验，发现在极高训练规模下，用 Adam 预训练的 ResNet152x2 在 ImageNet 微调达到了 84.97%，而传统的 SGD 只有 84.37%。这提示我们：当你把训练规模（Batch size、数据量）推向极端时，你过去深信不疑的经验法则（Rule of Thumb）可能需要重新审视。

此外，作者尝试了类似 BERT 里的 Masked Patch Prediction 进行自监督预训练，虽然取得了 79.9%（较从头训练有 2% 提升），但离监督学习还有 4% 的差距。这个“半成品”的坑，在一年后被何恺明大神用极简的 MAE 架构彻底填平，完成了大一统拼图的最后一块。
