ViT 证明了 Transformer 能做图像分类，但它无法取代 CNN 成为通用的视觉骨干网络——全局自注意力的平方级复杂度使其无法处理高分辨率输入，单一尺度的特征图也无法对接检测、分割等下游任务。2021 年，微软亚洲研究院提出的 <strong>Swin Transformer</strong> 解决了这两个核心问题，凭借层级式特征图和移动窗口机制一举拿下 ICCV Best Paper，成为首个在检测和分割任务上全面碾压 CNN 的 Transformer 架构。

## 研究动机

将 Transformer 直接应用于视觉任务面临两大根本挑战：

<strong>尺度变化巨大</strong>：图像中的物体尺寸差异很大。目标检测和实例分割等任务极度依赖类似 FPN（特征金字塔）的多尺度特征来识别不同大小的物体。而 ViT 生成的特征图是固定分辨率的（始终 16x 下采样），无法提供多尺度信息。

<strong>平方级计算复杂度</strong>：ViT 的全局自注意力计算复杂度为 $O(N^2)$，其中 $N$ 是 Patch 数量。当处理高分辨率图像（如语义分割常用的 $512 \times 512$ 甚至更大）时，计算量和内存消耗爆炸式增长。

<strong>Swin Transformer 的核心思路</strong>：设计一种具有层级结构且计算复杂度与图像大小呈<strong>线性关系</strong>的 Transformer，使其能像 ResNet 一样无缝接入各种下游密集预测任务。

| 特性 | ViT | Swin Transformer |
|------|-----|------------------|
| 自注意力范围 | 全局 | 局部窗口 + 移动窗口 |
| 计算复杂度 | $O((H \times W)^2)$ | $O(H \times W)$ |
| 特征图尺度 | 单一尺度 | 多尺度层级（类似 CNN） |
| 下游任务 | 主要分类 | 通用 backbone（分类/检测/分割） |

## 整体架构

![Swin Transformer 层级架构：4 个 Stage 逐步降低分辨率、增加通道数，Patch Merging 连接相邻 Stage](/static/blog/swin-transformer/architecture.svg)

Swin Transformer 的整体设计与 CNN 十分相似，分为四个 Stage，逐步降低分辨率、增加通道数：

### Patch Partition 与 Linear Embedding

将输入 $224 \times 224 \times 3$ 图像切割成 $4 \times 4$ 大小的不重叠 Patch：

- 切割后 Patch 数量：$(224/4)^2 = 56 \times 56 = 3136$ 个
- 每个 Patch 维度：$4 \times 4 \times 3 = 48$
- 线性层映射到 $C = 96$（Swin-T 配置）
- 输出形状：$56 \times 56 \times 96$

由于 3136 个 Token 做全局自注意力开销过大，Swin 引入窗口机制，每个窗口默认包含 $7 \times 7 = 49$ 个 Patch。

### 四个 Stage 与 Patch Merging

每个 Stage 内，Swin Transformer Block 不改变输入输出尺寸。Stage 之间通过 <strong>Patch Merging</strong> 逐步降低分辨率：

$$
56 \times 56 \times 96 \rightarrow 28 \times 28 \times 192 \rightarrow 14 \times 14 \times 384 \rightarrow 7 \times 7 \times 768
$$

Patch Merging 的具体步骤（类似 Pixel Shuffle 的逆过程）：

1. 对张量每隔一个点采样，得到四个 $H/2 \times W/2$ 的子张量
2. 按通道维度拼接，得到 $H/2 \times W/2 \times 4C$ 的张量
3. 通过线性变换将通道数降为 $2C$

最后经过 Average Pooling 变为 $1 \times 768$，再接分类头输出。这种金字塔式的多尺度特征图可以直接输入 FPN 做检测、输入 U-Net 做分割。

## 窗口自注意力（W-MSA）

### 复杂度分析

标准全局自注意力的复杂度：

$$
\Omega(\text{MSA}) = 4hwC^2 + 2(hw)^2C
$$

Swin 将注意力计算限制在 $M \times M$ 大小的窗口内（默认 $M = 7$），共有 $hw / M^2$ 个窗口：

$$
\Omega(\text{W-MSA}) = 4hwC^2 + 2M^2hwC
$$

当 $M$ 固定时，复杂度关于 $hw$ 为<strong>线性</strong>。以 Swin-T 第一阶段（$56 \times 56 \times 96$）为例，窗口注意力的 FLOPs 仅为全局注意力的约 1/8。

![全局 MSA（红）vs 窗口 W-MSA（蓝）的计算复杂度对比：窗口注意力随 Token 数线性增长](/static/blog/swin-transformer/complexity-comparison.png)

### 相对位置偏置

Swin 摒弃了 ViT 的绝对位置编码，在注意力计算中加入<strong>相对位置偏置</strong> $B$：

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}} + B\right) V
$$

窗口内沿两个坐标轴的相对位置范围为 $[-(M-1), M-1]$，模型参数化一个 $(2M-1) \times (2M-1)$ 的偏置表，$B$ 中的值通过坐标索引查表得到。消融实验表明，相对位置偏置比绝对位置编码在密集预测任务上效果显著更好。

## 移动窗口机制（SW-MSA）

### 为什么需要移动窗口？

固定窗口内自注意力虽然高效，但窗口之间完全没有信息交互。移动窗口机制使得原本只能在同一窗口内交互的 Patch 可以与相邻窗口的 Patch 进行自注意力计算。

### Swin Transformer Block

每个 Block 由两个连续子块组成，交替使用 W-MSA 和 SW-MSA：

$$
\hat{z}^l = \text{W-MSA}(\text{LN}(z^{l-1})) + z^{l-1}
$$

$$
z^l = \text{MLP}(\text{LN}(\hat{z}^l)) + \hat{z}^l
$$

$$
\hat{z}^{l+1} = \text{SW-MSA}(\text{LN}(z^l)) + z^l
$$

$$
z^{l+1} = \text{MLP}(\text{LN}(\hat{z}^{l+1})) + \hat{z}^{l+1}
$$

### 循环移位实现

移动窗口并非真的移动"框"，而是通过<strong>循环移位</strong>（Cyclic Shift）高效实现：

1. <strong>循环移位</strong>：将特征图向左上方移动 $\lfloor M/2 \rfloor$ 个 Patch（`torch.roll`）
2. <strong>重新划分窗口</strong>：在移位后的特征图上正常划分，窗口数保持不变
3. <strong>掩码注意力</strong>：移位后同一窗口可能包含不相邻区域，通过掩码矩阵（不相邻区域填 $-100$）屏蔽非法交互，$\text{softmax}(A + \text{Mask})$ 后对应位置权重趋近于 0
4. <strong>逆循环移位</strong>：计算完成后移回原位

![窗口划分、循环移位与掩码注意力的可视化](/static/blog/swin-transformer/window-cyclic-shift.png)

这种设计避免了直接移动窗口导致的碎片窗口问题（2x2 变成 3x3 个不等大小窗口），消融实验表明比 Naive Padding 做法快 <strong>13%-18%</strong>。

### COCO 目标检测

Swin 在检测任务上的表现震惊了整个 CV 界。在同等参数和 FLOPs 下，使用 Cascade Mask R-CNN 框架，<strong>Swin-T 比 ResNet-50 的 Box AP 暴涨 +4.2，Mask AP 暴涨 +3.3</strong>。最终融合 HTC++ 策略的 Swin-L 拿下 <strong>58.7 Box AP</strong>，实现了对纯 CNN 架构的跨代碾压。

### 关键消融实验

- <strong>Shifted Window</strong>：取消 Shift 后，ImageNet Top-1 掉 1.1%，COCO AP 暴跌 2.8 个点——跨窗口信息交互是重建感受野的关键
- <strong>相对位置偏置</strong>：加入绝对位置编码反而使密集任务性能下降；相对位置偏置比无位置编码高 1.2%

## 讨论与展望

### 创新之处

- <strong>完美的架构对齐</strong>：借用了 CNN "逐渐降采样"的智慧，完美适配 FPN、U-Net 等下游框架，工程友好度满分
- <strong>优雅的复杂度解决路径</strong>：Local Window 控制复杂度为 $O(N)$，Shift 实现跨窗口信息交互，设计哲学清晰
- <strong>断层级的霸榜性能</strong>：在检测和分割任务上以 2.0+ AP 的优势碾压所有纯 CNN 架构

### 局限性

- <strong>局部性妥协</strong>：Swin 重新走回了"通过堆叠局部感受野来扩大全局感受野"的老路，损失了 ViT 第一层就能建立全局关联的核心红利
- <strong>手工设计痕迹</strong>：窗口大小固定 $M=7$、Shift 位移固定 $\lfloor M/2 \rfloor$、Patch Merging 的 $2 \times 2$ 拼接——本质上和 CNN 的 Kernel Size=3x3 并无哲学区别。Swin 更像是"披着 Self-Attention 外衣的强悍 CNN"

### 实现 A：从零手写 Swin Transformer

逐组件实现适配 CIFAR-10（32x32）的小型 Swin Transformer：

核心的窗口注意力 + 相对位置偏置实现：

```python
class WindowAttention(nn.Module):
    def __init__(self, dim, window_size, num_heads):
        super().__init__()
        self.scale = (dim // num_heads) ** -0.5
        # 相对位置偏置表: (2M-1)^2 x num_heads
        self.relative_position_bias_table = nn.Parameter(
            torch.zeros((2*window_size-1)**2, num_heads))
        # 预计算相对位置索引
        coords = torch.stack(torch.meshgrid(
            torch.arange(window_size),
            torch.arange(window_size), indexing='ij'))
        relative_coords = coords[:,:,None] - coords[:,None,:]
        # ... 偏移到非负并展平为查表索引
        self.qkv = nn.Linear(dim, dim * 3)

    def forward(self, x, mask=None):
        qkv = self.qkv(x)  # (nW*B, M*M, 3C)
        q, k, v = ...       # 分头
        attn = (q @ k.T) * self.scale + relative_position_bias
        if mask is not None:
            attn = attn + mask  # 移动窗口掩码
        return softmax(attn) @ v
```

循环移位实现 SW-MSA 的关键代码：

```python
# 循环移位
shifted_x = torch.roll(x, shifts=(-shift, -shift), dims=(1, 2))
# 窗口划分 + 注意力计算（带掩码）
x_windows = window_partition(shifted_x, window_size)
attn_windows = self.attn(x_windows, mask=attn_mask)
# 逆循环移位
x = torch.roll(shifted_x, shifts=(shift, shift), dims=(1, 2))
```

### 实现 B：torchvision 预训练微调

使用 `torchvision.models.swin_t` 加载 ImageNet 预训练权重，仅微调分类头：

```python
from torchvision.models import swin_t, Swin_T_Weights

model = swin_t(weights=Swin_T_Weights.IMAGENET1K_V1)
model.head = nn.Linear(model.head.in_features, 10)  # CIFAR-10

# 冻结骨干，仅训练分类头
for name, param in model.named_parameters():
    if 'head' not in name:
        param.requires_grad = False
```
